## 4. Key Gaps and Recommendations

### 4.1 Identified Gaps

| Gap | Severity | Description |
|-----|----------|-------------|
| G1 | Medium | D3 (no-op detection) not strictly enforced |
| G2 | Low | `parent_updates` in MERGE_SUBTASK_RESULT not validated |
| G3 | Low | `allowed_evidence_types` not strictly validated |
| G4 | Low | No `CANCEL_TASK` / `REPLAN_SUBTASKS` actions |

### 4.2 Recommendations

#### G1: Implement Strict No-Op Detection

**Current**: LoopDriver tracks state but doesn't reject no-op actions.

**Recommendation**: Add `validateStateChange()` method:

```typescript
// In validator.ts
function validateStateChange(action: OrchestratorAction, oldState: OrgState, newState: OrgState): ValidationResult {
  if (JSON.stringify(oldState) === JSON.stringify(newState)) {
    return {
      ok: false,
      errors: [{
        code: "NO_STATE_CHANGE",
        message: "Action did not change observable state",
        path: "root"
      }]
    };
  }
  return { ok: true, errors: [], warnings: [] };
}
```

#### G2: Validate MERGE parent_updates

**Recommendation**: Add validation in `citation-validator.ts`:

```typescript
// Validate parent_updates fields are properly formatted
if (payload.parent_updates) {
  // Validate findings_append format
  // Validate next_actions_append format
}
```

#### G3: Strict Evidence Type Validation

**Recommendation**: Enhance `role-gate-validator.ts`:

```typescript
// In validatePhaseGate()
if (gate.allowed_evidence_types && !gate.allowed_evidence_types.includes(evidence.type)) {
  errors.push({
    code: "EVIDENCE_TYPE_NOT_ALLOWED",
    message: `Evidence type '${evidence.type}' not allowed for this gate`,
  });
}
```
